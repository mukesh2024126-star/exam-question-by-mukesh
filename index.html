1) Compression (Huffman, LZW, ASCII)

SolvedApply LZW compression to ABABABAB. Write output codes.
Output sequence (ASCII initialized, next code 256): <65><66><256><258><66>.
Added dictionary entries: AB=256, BA=257, ABA=258, ABAB=259.
Solved"advantage" encoded with Huffman length = 24 bits, ASCII uses 8 bits/char. Find original bits, compressed bits, compression ratio.
Word length = 9 characters.
Original bits = 9 × 8 = 72 bits
Compressed bits = 24 bits
Compression ratio = 72 / 24 = 3:1
Saving = ((72−24)/72)×100 = 66.67%
Solved"advantage" with LZW: original 64 bits, compressed 32 bits. Find ratio.
Compression ratio = 64/32 = 2:1; saving = 50%.
SolvedOriginal size 200 KB, compressed 50 KB. Find compression ratio.
Compression ratio = 200/50 = 4:1; saving = 75%.
SolvedIf original size = 72 bits and compressed size = 24 bits, find compression percentage.
Saving % = ((72−24)/72)×100 = 66.67%. Ratio = 3:1.
SolvedFor 2000 characters with average 3 bits/symbol, find total bits.
Total bits = 2000 × 3 = 6000 bits.
MCQIn Huffman encoding, binary length depends on ______ of character.
Frequency/probability of the character.
MCQMain advantage/purpose of Huffman coding?
Variable-length lossless coding where frequent symbols get shorter codes, reducing average bits.
MCQMain purpose/use of LZW compression?
Lossless dictionary-based compression of repeated sequences; used in formats like GIF/TIFF.
MatchMatch Huffman, LZW, ASCII, Edge encoding.
Huffman → variable-length frequency-based coding
LZW → dictionary-based coding
ASCII → fixed-length character coding (8-bit in exam problems)
Edge encoding → edge/structure-focused image coding concept

3) Histogram, Brightness, Contrast (completed incomplete questions)

“Histogram moves to ______ when ______ to lose image information.”
Moves right when brightness increases; moves left when brightness decreases. If shifted beyond [0,255], clipping occurs and detail is lost.
Main purpose of histogram?
To show intensity distribution and analyze brightness, contrast, dynamic range, and clipping.
What changes in histogram when image detail is lost?
Spikes/pile-up near 0 or 255 (clipping), reduced meaningful tonal spread.
In image processing, analyzing histogram shows ______.
Brightness and contrast distribution of the image.
“To ______ an RGB image when difference between most & least intensity value.”
To increase contrast, increase the difference between max and min intensity values.
“To ______ of RGB color of most and least color while brightness remains unchanged.”
To adjust saturation, modify max–min RGB difference while preserving brightness/luma.

4) Spatial Filtering & Edge Detection

Purpose of spatial filtering?
Neighborhood-based processing for smoothing, sharpening, edge detection, and feature extraction.
Explain Canny edge detection.
Multi-stage method: Gaussian smoothing → gradient → non-maximum suppression → double threshold/hysteresis.
TopicExplain Gaussian filter (separate question).
Gaussian is a smoothing/low-pass filter. It reduces random noise and fine detail before operations like edge detection.
TopicExplain Laplacian filter (separate question).
Laplacian is a second-derivative high-pass operator. It responds strongly to rapid intensity changes (edges/fine detail), but can also amplify noise.
TopicExplain convolution (separate question).
Convolution computes each output pixel as a weighted sum of neighborhood pixels using a kernel after spatial flipping. For symmetric kernels, convolution and correlation give the same values.
MCQWhich filter is best before Canny edge detection?
Gaussian smoothing, to suppress noise and reduce false edges.
MCQWhich operator is based on second derivative?
Laplacian.
MCQIn convolution, what is the key difference from correlation?
The kernel is flipped in convolution.
MatchGaussian / Laplacian / Convolution — match the following.
Gaussian → Noise reduction / smoothing
Laplacian → Edge/detail enhancement (2nd derivative)
Convolution → Kernel-based neighborhood weighted sum (with kernel flip)
“______ central pixel using both correlation/convolution.”
Compute processed central pixel as weighted sum of neighborhood values using the filter kernel.
“Sharpened image using Laplacian is obtained by ______.”
Subtracting Laplacian-filtered output from original image:
g = f − ∇²f
Egg crack detection with backlight — which filter?
Edge/high-pass filtering (commonly Laplacian or Canny-style edge detector).

5) RGB, Color Models, Display

In LCD, why RGBG used instead of RGB?
Human vision is more sensitive to green/luminance detail, so extra green improves perceived sharpness/efficiency.
What is the difference between hue, saturation, brightness?
Hue = color type; Saturation = color purity/vividness; Brightness/Luma = lightness perception.
Best grayscale/luma estimate from RGB?
Y = 0.3R + 0.59G + 0.11B
If R=G=B, what does it mean?
Neutral grayscale; saturation is 0.
